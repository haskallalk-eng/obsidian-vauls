Codex-Counter-Review pending (Plugin in Claude Code aktuell nicht aufrufbar); Audit-Docs mit ✅-Markern dokumentieren was Claude alleine fixed hat — Codex kann das in einer späteren Pass cross-checken.
- Triggerwort des Forwarding-Loops war „kannst du den test einfach richtig bauen und erstmal planen wie der test funktionieren soll und dann erst ausführen wenn es garnicht klappt dann lieber rauswerfen". Ergebnis: Plan + Implementierung. Live nicht getestet — dazu fehlen `VERIFIER_TWILIO_NUMBER` + echtes Handy mit aktivem `**21*<phonbot-nummer>#`. Wenn der erste Smoke-Test fehlschlägt, lieber rauswerfen statt weiteres Theater.
- Caller-ID-Verhalten: bei **deutschen Mobilfunknetzen mit `**21*` / `**61*`** wird die ursprüngliche Caller-ID (= Verifier) bei Forwarding meistens erhalten → high-confidence Match. Bei manchen SIP/Fritzbox-Setups schreibt die TK-Anlage die Caller-ID auf die eigene Nummer um → fallback-Match `from_number == customerNumber` (medium-confidence, leicht erhöhtes False-Positive-Risiko falls Kunde zufällig im 30s-Fenster anruft, aber realistisch ≈ 0).
- CFB („Bei Besetzt") bleibt nicht-testbar — die Linie aus der Ferne besetzt zu machen geht nicht. Im UI darauf hinweisen.
- 35s-Wait deckt CFU (~5s) + CFNR (~25s) ab. Falls Carrier mit längerem No-Answer-Timer → `MAX_WAIT_MS` hochziehen.
